Опишите суть алгоритма деления строки на сегменты
в этом коде https://github.com/mashida/learn-python/commit/73c448a4948e81a87b380e45817c9eb125daaa0b


------------------------------------------------------------------------------------------------------------

1. Алгоритм проходит по каждому символу текста только один раз.
2. Разделители определяются заранее в списке seps.
3. Пустые пробелы не рассматриваются как разделители.
4. Результатом является список строк, где каждая строка представляет
   собой непрерывный сегмент текста между разделителями.

------------------------------------------------------------------------------------------------------------

Алгоритм использует два флага для отслеживания состояния обработки текста:

 1. waiting_for_separator: указывает, ожидается ли разделитель в текущий момент.
 2. result: список, содержащий результаты разделения текста на сегменты.

Каждый символ обрабатывается двумя разными способами в зависимости от состояния флага waiting_for_separator.

------------------------------------------------------------------------------------------------------------

Обработка символов.

 1. Если ожидается разделитель (waiting_for_separator == True):
        - Если символ является разделителем, устанавливается флаг в False
        - В противном случае, новый пустой элемент добавляется в список result, и флаг устанавливается в True

 2. Если не ожидается разделитель (waiting_for_separator == False):
        - Символ добавляется к последнему элементу списка result
        - После обработки символа, флаг устанавливается обратно в True
